# Gemma Harness — Status Update (2026-04-22, session 2)

## What was found and fixed in this session

### New root cause identified: AgentTimeoutError

Investigation of the last EASY_SUBSET baseline run (`baseline_2026-04-21__12-09-05`)
revealed that **all 6 trials of `openssl-selfsigned-cert` and `sanitize-git-repo`
were killed by Harbor's 900s hard agent timeout** — not by protocol drift, not by
model incapability, and not by tool errors.

The previous session left `max_turns: 40` in `config.yaml`. On CPU-only serving
(ROCm failed to init) at ~50s/turn, that's a 2000s theoretical budget against a
900s hard cap. Every single trial hit the wall.

Full analysis: `docs/OBSERVED_ISSUE_2026-04-22_AGENT_TIMEOUT.md`

### Fixes applied (commits `3f83f06` and `5d6175b`)

| File | Change | Reason |
|---|---|---|
| `harness/harness.py` | Retry-with-repair (A.5) | Self-correction on drift-in-content turns |
| `scripts/record_baseline.py` | `repair_attempts` in ledger | Track repair events |
| `config.yaml` | `max_turns`: 40 → 20 | All wins in ≤11 turns; fits in timeout |
| `config.yaml` | `model_timeout_sec`: 90 → 60 | Tighter per-call cap, CPU median ~12s |
| `scripts/baseline.sh` | `--agent-timeout-multiplier 3.0` | 2700s effective cap |
